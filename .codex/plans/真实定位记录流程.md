# 真实定位接入记录流程

## 目标
进入环境页时优先尝试一次性真实定位，成功后用真实坐标和反地理编码地点写入记录；失败时降级到现有 Mock 坐标和可手动编辑地点，WeatherKit 暂不接入。

## 实施计划
1. 阅读 `EnvDataView`、`RecordSession`、`WeatherService` 相关实现。✅
2. 调整环境页加载流程：真实定位优先，失败后 Mock 兜底。✅
3. 保持天气服务 Mock，仅将传入坐标改为真实钓点坐标。✅
4. 运行可用构建 / 测试检查。✅
5. 自审 diff，提交并创建中文 PR。✅

## 风险与取舍
- 定位权限拒绝、模拟器无定位时必须不阻塞记录流程。
- 不新增 WeatherKit 和外部依赖，降低签名与权限风险。
- 仅修改记录环境页，避免扩展到历史详情或分享链路。
- `WeatherService` 仍返回 Mock 数据，本次只确保天气加载使用真实钓点坐标作为后续 WeatherKit 接入基础。
- 为避免模拟器或权限弹窗长时间无回调，单次定位增加 8 秒超时兜底。

## 验证结果
- `git diff --check` 通过。
- `xcodebuild -project FishingDiary.xcodeproj -scheme FishingDiary -destination 'generic/platform=iOS Simulator' -quiet build` 未执行成功：当前环境没有 `xcodebuild`。

## 结果复盘
- 环境页加载流程已从固定 Mock 坐标调整为真实定位优先。
- 定位失败或反地理编码失败时仍保留 Mock 坐标 / 地名兜底，不阻塞用户保存日记。
- 未接入 WeatherKit，符合本次低风险范围。
