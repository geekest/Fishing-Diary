# 多语言切换执行计划

## 目标

在设置页新增「语言」入口，进入二级页面后支持切换简体中文、English、日语，并在切换后立即生效。

## 初始方案

1. 阅读设置页、根视图、App 入口和现有文案组织方式。
2. 选择最小实现：优先使用 SwiftUI 环境 `locale` + AppStorage 保存语言选择；若项目缺少本地化资源，则补充必要 `Localizable.strings`。
3. 在设置页新增语言入口和语言选择二级页面。
4. 验证构建或说明环境限制。
5. 提交 commit 并创建中文 PR。

## 风险与取舍

- 如果现有页面文案大量硬编码，本次仅覆盖主要 App 文案和新增语言功能，避免一次性大范围重构。
- SwiftUI `Text("key")` 可随 `locale` 切换即时刷新；纯 `String` 拼接或系统组件标题可能需要逐步补齐。
